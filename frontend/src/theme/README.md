# Theme System Guide

This folder contains the centralized theme system for the Expo app.

The goal is:
- Keep **all colors, gradients, typography, radii, spacing** in one place.
- Reuse **shared UI components** (buttons/cards/text) so the app looks consistent.
- Support **Light/Dark mode** from the same theme tokens.

## Folder map

- `palette.ts`
  - Base color palette (raw colors)
  - You generally *reference* these in `themes.ts`

- `themes.ts`
  - The main place to change the UI
  - Exports:
    - `lightTheme`
    - `darkTheme`
    - `AppTheme` type

- `ThemeProvider.tsx`
  - Chooses light/dark based on device setting (`useColorScheme()`)
  - Provides theme via React Context

- `useTheme.ts`
  - Hook used by screens/components to read theme

## Where to change what

### 1) Background gradient
Edit `themes.ts`:
- `lightTheme.colors.background.gradient`
- `darkTheme.colors.background.gradient`

Example:
- Light: `['#c7d2fe', '#e9d5ff', '#fecdd3']`
- Dark: `['#0b1220', '#180b2a', '#2b0b17']`

### 2) Text colors
Edit `themes.ts`:
- `theme.colors.text.primary`
- `theme.colors.text.secondary`
- `theme.colors.text.muted`
- `theme.colors.text.link`
- `theme.colors.text.inverse`

Usage guideline:
- Use `primary` for headings and important text
- Use `secondary` for normal paragraphs
- Use `muted` for helper text, subtitles, captions
- Use `link` for highlighted text (brand color)

### 3) Glass/Surface colors (cards, header badge backgrounds)
Edit `themes.ts`:
- `theme.colors.surface.glass` (used by `ThemedCard` and header language badge)
- `theme.colors.surface.footer` (optional; if you add a fixed footer later)

### 4) Borders
Edit `themes.ts`:
- `theme.colors.border.hairline` (thin separators)
- `theme.colors.border.outline` (outline button border)

### 5) Brand color
Edit `themes.ts`:
- `theme.colors.brand.primary`

This is used in places like:
- highlighted title word "Grow."
- feature card icon background (first card)

### 6) Buttons (global)
Buttons are centralized in:
- `src/components/ThemedButton.tsx`

#### Change colors (recommended)
Edit `themes.ts`:
- `theme.colors.button.primaryBg`
- `theme.colors.button.primaryText`
- `theme.colors.button.outlineText`
- `theme.colors.button.outlineBorder`

#### Change sizes/radius
Edit `themes.ts`:
- `theme.radii.button`

Edit `ThemedButton.tsx` if you want to change default height:
- `height: 56`

### 7) Cards (global)
Cards are centralized in:
- `src/components/ThemedCard.tsx`

Change defaults in:
- `theme.colors.surface.glass`
- `theme.radii.card`

### 8) Typography (sizes/weights)
Edit `themes.ts`:
- `theme.typography.title` (hero title)
- `theme.typography.headerTitle` (app title in header)
- `theme.typography.headerSubtitle` (header subtitle)
- `theme.typography.body`
- `theme.typography.button`

Important:
- Currently the app uses only **fontWeight**. There is no custom font family loaded yet.

## Adding a custom font family (recommended)

If you want a single font (example: Inter / Poppins) across the entire app:

1) Install:
- `npx expo install expo-font`

2) Add font files under something like:
- `src/assets/fonts/`

3) Load fonts once at app startup (in `AppRoot`), then:
- Add `theme.typography.fontFamily.*` tokens (we can extend `AppTheme`)
- Update `ThemedText` and `ThemedButton` to apply the default font family

If you tell me the font name you want, I can wire this for you.

## Light/Dark mode behavior

- The theme is chosen automatically using system settings:
  - `ThemeProvider.tsx` uses `useColorScheme()`

To force a manual toggle later:
- Store a preference in state / AsyncStorage
- Replace the `useColorScheme()` decision with user preference

## How to build new screens (rules)

- Prefer:
  - `ThemedText` instead of `Text`
  - `ThemedButton` instead of custom `Pressable` buttons
  - `ThemedCard` for any card surfaces

- Avoid hardcoding colors like `#0f172a` inside screens.
  - Put it in `themes.ts` and reference it via the theme.

- Keep spacing/radius consistent:
  - Use `theme.spacing.screenX` for horizontal screen padding
  - Use `theme.radii.*` for radius values

## Quick example usage

```ts
import { useTheme } from "../theme/useTheme";
import { ThemedText } from "../components/ThemedText";
import { ThemedButton } from "../components/ThemedButton";

export function MyScreen() {
  const theme = useTheme();

  return (
    <>
      <ThemedText style={{ fontSize: 20, fontWeight: "800" }}>Hello</ThemedText>
      <ThemedText variant="muted">Subtitle</ThemedText>

      <ThemedButton title="Continue" variant="primary" />
      <ThemedButton title="Cancel" variant="outline" />

      <ThemedText style={{ color: theme.colors.brand.primary }}>Brand text</ThemedText>
    </>
  );
}
```
